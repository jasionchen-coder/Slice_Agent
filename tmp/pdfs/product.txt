直播切片 Agent 产品需求文档 PRD
1. 文档信息
产品名称： 直播切片 Agent
文档版本： V1.0
产品类型： AI 视频内容处理工具
核心目标： 系统自动理解直播回放内容，并自动生成可发布的短视频切片。
适用场景： 电商直播、知识直播、访谈直播、课程直播、游戏直播、活动直播。




2. 产品背景
很多直播内容时长较长。用户很难手动从完整直播回放中找到有价值的片段。人工切片需要反复观看视频。这
个过程成本高，效率低，也容易遗漏重点内容。


本产品希望通过 AI Agent 自动完成直播回放分析。系统会先加载直播回放视频。然后系统会提取音频。接着系
统会把音频转成文字。然后大模型会理解文字内容。最后系统会根据内容重点、话题边界、情绪变化和可传播
价值，对完整视频进行自动切片。




3. 产品目标

3.1 核心目标

  1. 用户上传或选择直播回放视频后，系统可以自动生成多个视频切片。
  2. 每个切片都要包含开始时间、结束时间、标题、摘要和推荐理由。
  3. 用户可以预览、修改和导出切片结果。
  4. 系统要尽量减少人工观看完整视频的时间。

3.2 业务目标

  1. 提高直播内容二次分发效率。
  2. 降低人工剪辑成本。
  3. 帮助用户快速发现直播中的高价值片段。
  4. 为短视频平台、私域内容和营销素材提供内容来源。




4. 产品范围

4.1 本期范围

本期只处理直播回放视频。系统不处理实时直播流。


本期包含以下能力：


  1. 视频导入。

   2. 音频提取。
   3. 语音转文字。
   4. 文字分段。
   5. 大模型内容理解。
   6. 自动生成切片方案。
   7. 根据时间戳切割视频。
   8. 切片结果预览。
   9. 切片结果导出。
  10. 切片任务状态展示。

4.2 暂不包含范围

本期暂不处理以下能力：


  1. 实时直播切片。
  2. 自动发布到抖音、快手、小红书、B站等平台。
  3. 视频自动配字幕样式。
  4. 视频自动封面设计。
  5. 视频自动混剪。
  6. 多机位视频处理。
  7. 复杂版权审核。
  8. 主播数字人二次生成。




5. 用户角色

5.1 内容运营人员

内容运营人员希望快速从直播回放中找到可传播片段。运营人员更关注切片标题、传播价值和导出效率。


5.2 视频剪辑人员

视频剪辑人员希望系统先生成初步切片方案。剪辑人员可以在系统结果上进行二次编辑。


5.3 直播团队负责人

直播团队负责人希望通过切片结果复盘直播内容。负责人更关注哪些内容值得复用，哪些内容有转化价值。




6. 核心使用流程

6.1 用户流程

  1. 用户进入系统前端。
  2. 用户上传直播回放视频，或选择已有视频。
  3. 系统展示视频基础信息。
  4. 用户点击“开始分析”。
  5. 系统提取视频音频。
  6. 系统把音频转成文字。
  7. 系统把文字交给大模型进行内容分析。

  9. 系统按照切片建议切割视频。

 11. 用户预览每个切片。
 12. 用户修改切片标题、摘要、开始时间和结束时间。
 13. 用户导出最终切片视频。

7.5 大模型内容分析模块

功能说明

系统把带时间戳的文字稿交给大模型。大模型需要理解直播内容，并判断哪些片段适合切成短视频。


分析维度

大模型需要重点分析以下内容：


     1. 话题边界： 判断一个完整话题从哪里开始，到哪里结束。
     2. 内容价值： 判断片段是否有信息量、观点、卖点或故事性。
     3. 传播价值： 判断片段是否适合做短视频传播。
     4. 情绪变化： 判断是否出现高情绪、高互动或高冲突内容。
     5. 商品卖点： 如果是电商直播，系统要识别商品名称、价格、优惠和卖点。
     6. 知识点： 如果是课程直播，系统要识别重要知识点。
     7. 风险内容： 系统要标记可能不适合发布的内容。

验收标准

         1. 大模型必须输出结构化 JSON。
         2. 每个切片必须包含开始时间和结束时间。
         3. 每个切片必须包含标题、摘要和推荐理由。
         4. 每个切片的开始时间必须小于结束时间。
         5. 每个切片的时长必须在系统设定范围内。
         6. 大模型输出失败时，系统需要自动重试或进入人工处理状态。




7.6 切片策略模块

功能说明

系统根据大模型分析结果，生成最终切片方案。


默认切片规则

         1. 单个切片建议时长为 30 秒到 180 秒。
         2. 切片内容要尽量表达一个完整话题。
         3. 切片开始时间可以向前扩展 2 到 5 秒。
         4. 切片结束时间可以向后扩展 2 到 5 秒。
         5. 切片之间允许有少量重叠。
         6. 系统需要过滤信息量过低的片段。
         7. 系统需要过滤风险等级较高的片段。

切片评分规则

每个切片需要生成一个综合分数。满分为 100 分。


                       评分项      权重    说明

                       内容完整度    30%   片段是否有清楚的开头和结尾

                       信息密度     25%   片段是否包含有效信息

                       传播价值     25%   片段是否适合短视频传播

                       情绪强度     10%   片段是否有情绪、冲突或互动

                       风险控制     10%   片段是否适合对外发布


验收标准

         1. 系统可以根据评分排序切片。
         2. 系统默认优先展示高分切片。
         3. 系统可以过滤低分切片。
         4. 系统可以限制最大切片数量。

 请你完成以下任务：
 1. 识别每个话题的开始时间和结束时间。
 2. 总结每个话题的核心内容。
 3. 判断每个话题是否适合切成短视频。
 4. 标记片段中的商品卖点、知识点、冲突点、情绪点和风险内容。
 5. 输出结构化 JSON。

10.2 切片判断 Prompt

 你是一个直播切片策略专家。你需要根据直播文字稿和话题分析结果，生成适合短视频传播的切片方案。

 请你遵守以下规则：
 1. 每个切片必须围绕一个完整主题。
 2. 每个切片的开始时间必须早于结束时间。
 3. 每个切片建议时长为 30 秒到 180 秒。
 4. 每个切片必须包含标题、摘要、推荐理由和评分。
 5. 不要选择信息量很低的闲聊片段。
 6. 不要选择风险等级为 high 的片段。
 7. 输出必须是合法 JSON。
 8. 不要输出 JSON 以外的解释文字。

11. 非功能需求

11.1 性能需求
     1. 系统需要支持至少 1 小时直播回放处理。
     2. 系统需要支持后台异步任务处理。
     3. 系统需要展示任务进度。
     4. 系统不能因为单个任务失败导致整体服务不可用。
     5. 系统需要支持多个任务排队处理。


11.2 稳定性需求
     1. 视频上传失败时，系统需要支持重新上传。
     2. ASR 失败时，系统需要支持重试。
     3. 大模型输出格式错误时，系统需要支持自动修复或重新请求。
     4. 视频切割失败时，系统需要保留错误日志。
     5. 系统需要保存每一步的中间结果。


11.3 安全需求
     1. 系统需要限制上传文件大小。
     2. 系统需要校验文件类型。
     3. 系统需要避免用户上传恶意文件。
     4. 系统需要保护原始视频文件和切片文件。
     5. 系统需要设置任务访问权限。


11.4 可扩展需求
     1. 后续可以支持实时直播切片。
     2. 后续可以支持自动生成字幕。
     3. 后续可以支持自动生成封面。
     4. 后续可以支持自动发布到内容平台。
     5. 后续可以支持多语言直播识别。
     6. 后续可以支持多角色说话人分析。

13. 关键指标

14. MVP 版本建议
MVP 版本建议优先完成以下功能：


   1. 视频上传。
   2. 音频提取。
   3. 语音转文字。
   4. 带时间戳文字稿生成。
   5. 大模型生成切片方案。
   6. 视频自动切割。
   7. 切片列表展示。
   8. 切片预览和下载。

MVP 版本暂时不需要做自动发布、自动封面和复杂字幕样式。系统先验证核心链路是否跑通。核心链路是：视

15. 验收标准

15.1 核心链路验收

  1. 用户上传一个直播回放视频后，系统可以正常创建任务。
  2. 系统可以从视频中提取音频。
  3. 系统可以生成带时间戳的文字稿。
  4. 系统可以调用大模型生成切片方案。
  5. 系统可以根据时间戳切割视频。
  6. 用户可以在前端预览切片。
  7. 用户可以下载切片文件。

15.2 切片结果验收

  1. 每个切片必须有标题。
  2. 每个切片必须有摘要。
  3. 每个切片必须有开始时间和结束时间。
  4. 每个切片必须有推荐理由。
  5. 每个切片必须有评分。
  6. 切片内容要和原视频内容匹配。
  7. 切片时长要符合系统配置。
  8. 高风险内容默认不进入推荐列表。




16. 后续迭代方向

16.1 第二阶段

  1. 支持字幕自动生成。
  2. 支持封面自动生成。
  3. 支持切片标题多版本生成。
  4. 支持用户选择切片风格。
  5. 支持切片结果人工标注。

16.2 第三阶段

  1. 支持实时直播切片。
  2. 支持自动发布到短视频平台。
  3. 支持根据平台规则生成不同版本。
  4. 支持账号级内容效果分析。
  5. 支持基于历史数据优化切片策略。




17. 关键说明
本产品的核心不是简单切割视频。它的核心是让系统先理解直播内容，再根据内容价值生成切片方案。




                         16
所以系统必须保证三件事：


  1. 文字稿必须带准确时间戳。
  2. 大模型输出必须是结构化结果。
  3. 最终视频切片必须和文字时间戳严格对应。

只要这三点稳定，直播切片 Agent 的 MVP 就可以成立。
